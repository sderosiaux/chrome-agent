LLM USAGE GUIDE:
  Workflow: inspect -> read uids -> act (click/fill) -> inspect again.
  Use --json for structured output. Headless by default.

  Quick start:
    aibrowsr goto https://example.com --inspect
    aibrowsr click e3 --inspect
    aibrowsr fill --uid e2 "hello@example.com"
    aibrowsr eval "document.title"

  Targeting elements (3 modes):
    aibrowsr click e4                              # by uid from inspect
    aibrowsr click --selector "button.submit"      # by CSS selector
    aibrowsr click --xy 100,200                    # by coordinates
    aibrowsr fill --uid e2 "value"                 # fill by uid
    aibrowsr fill --selector "input[name=q]" "val" # fill by CSS selector
    aibrowsr type "hello" --selector "input.search"  # type with focus

  Inspection:
    aibrowsr inspect                        # accessibility tree with uid=eN
    aibrowsr inspect --max-depth 2          # limit depth (saves tokens)
    aibrowsr inspect --uid e5               # subtree only
    aibrowsr text                           # visible page text (reader mode)
    aibrowsr text e5                        # text of specific element

  Navigation:
    aibrowsr goto https://example.com       # navigate
    aibrowsr back                           # history.back()
    aibrowsr scroll down                    # scroll page
    aibrowsr wait text "Success"            # wait for content

  Other:
    aibrowsr screenshot                     # returns file path (PNG)
    aibrowsr eval "JSON.stringify(...)"     # run JS, get result
    aibrowsr press Enter                    # press key
    aibrowsr hover e3                       # hover element
    aibrowsr fill-form e2="a" e3="b"        # batch fill by uid

  Multi-tab:
    aibrowsr --page tab2 goto https://other.com
    aibrowsr --page tab2 inspect

  JSON mode (for programmatic use):
    aibrowsr --json goto https://example.com
    -> {"ok":true,"url":"...","title":"..."}
    Errors: {"ok":false,"error":"...","hint":"..."} (exit 0)

  Tips:
    - inspect before interacting -- uids change when the page mutates.
    - --inspect on click/fill returns updated page state in one call.
    - Prefer inspect over screenshot (~50 vs ~100K tokens).
    - Use --selector when inspect uids aren't available or practical.
