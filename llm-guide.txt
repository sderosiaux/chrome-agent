LLM USAGE GUIDE:
  Workflow: inspect -> read uids -> act (click/fill) -> inspect again.
  Headless by default. Add --stealth for bot-protected sites. Use --json for structured output.

  Quick start:
    aibrowsr goto https://example.com --inspect
    aibrowsr click n12 --inspect
    aibrowsr fill --uid n20 "hello@example.com"
    aibrowsr read                              # article content (Readability)
    aibrowsr eval "document.title"

  Stealth (bypass Cloudflare/bot detection):
    aibrowsr --stealth goto https://protected-site.com --inspect
    # Patches navigator.webdriver, chrome.runtime, UA, WebGL, input leak, skips Runtime.enable

  Logged-in sites (X.com, Gmail, dashboards — use your Chrome cookies):
    aibrowsr --stealth --copy-cookies goto x.com/home --inspect
    # Copies cookies from your real Chrome profile. No manual login needed.

  Heavy bot protection (DataDome, Kasada — when --stealth isn't enough):
    # Use your real Chrome instead of bundled Chromium:
    google-chrome --remote-debugging-port=9222 &
    aibrowsr --connect http://127.0.0.1:9222 goto https://protected-site.com --inspect
    # Real Chrome passes fingerprint checks that Chromium can't

  Targeting elements (3 modes):
    aibrowsr click n40                              # by uid from inspect
    aibrowsr click --selector "button.submit"      # by CSS selector
    aibrowsr click --xy 100,200                    # by coordinates
    aibrowsr fill --uid n20 "value"                 # fill by uid
    aibrowsr fill --selector "input[name=q]" "val" # fill by CSS selector

  Inspection + filtering:
    aibrowsr inspect                               # full accessibility tree
    aibrowsr inspect --max-depth 2                 # limit depth (saves tokens)
    aibrowsr inspect --filter "button,link"        # only these roles (flat list)
    aibrowsr inspect --uid n50                     # subtree only

  Content extraction (choose the right tool):
    aibrowsr read                                  # article/main content (Readability)
    aibrowsr read --truncate 1000                  # capped at N chars
    aibrowsr text --selector "main" --truncate 500 # visible text of CSS-matched element
    aibrowsr text                                  # full page innerText (verbose)
    aibrowsr extract                                # auto-detect repeating data (tables, cards, lists)
    aibrowsr extract --scroll                      # scroll first for lazy-loaded pages (YouTube, Pinterest)
    aibrowsr eval "JSON.stringify(...)"            # structured data via JS
    aibrowsr eval --selector "h1" "el.textContent" # eval scoped to a CSS element (el = matched)
    aibrowsr network --filter "api" --body         # intercept API responses directly
    aibrowsr console --level error                 # see JS errors from the page

  Network capture ("Readability for APIs"):
    aibrowsr network                               # resources already loaded (stealth-safe)
    aibrowsr network --filter "graphql" --body     # filter + response bodies
    aibrowsr network --live 5 --body               # capture live traffic for 5 seconds

  Console capture (stealth-safe):
    aibrowsr console                               # all messages
    aibrowsr console --level error                 # errors + exceptions only

  Pipe mode (persistent connection, 10x faster):
    echo '{"cmd":"goto","url":"https://example.com","inspect":true}' | aibrowsr pipe
    # Returns one JSON line per command: {"ok":true,...}

  Navigation:
    aibrowsr goto https://example.com              # navigate
    aibrowsr goto https://spa.com --wait-for "main" # wait for selector after nav
    aibrowsr back                                  # history.back()
    aibrowsr scroll down                           # scroll page
    aibrowsr wait text "Success"                   # wait for content

  Multi-tab:
    aibrowsr --page tab2 goto https://other.com
    aibrowsr --page tab2 inspect

  Parallel agents (avoid session conflicts):
    aibrowsr --browser agent1 goto https://example.com  # isolated browser per agent
    aibrowsr --browser agent2 goto https://other.com    # separate Chrome instance

  JSON mode:
    aibrowsr --json goto https://example.com
    -> {"ok":true,"url":"...","title":"..."}
    Errors: {"ok":false,"error":"...","hint":"..."} (exit 0, always parseable)

  Cleanup:
    aibrowsr close                                 # stop browser
    aibrowsr close --purge                         # stop + delete cookies/profile

  Important tips:
    - Use --stealth on sites with bot protection (Cloudflare, Turnstile).
    - UIDs (n47, n123) are stable across inspects on the same page.
    - After SPA navigation (back, click that triggers route change), re-inspect — UIDs change.
    - For SPA product pages, prefer `goto <direct-url>` over `click` (avoids modal routing).
    - Use `read` for articles/blogs, `text --selector` for scoped extraction, `eval` for structured data.
    - Prefer inspect over screenshot (~50 vs ~100K tokens).
    - --max-depth works on goto/click/fill --inspect too (not just standalone inspect).
    - --filter gives a flat list — great for finding interactive elements quickly.
